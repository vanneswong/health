# 健康助手 (Health Buddy) - 需求文档

## 1. 项目概述

### 1.1 项目名称
健康助手 (Health Buddy)

### 1.2 项目简介
一个综合性健康管理Web应用，支持血压和血糖数据记录与分析，以及用药管理功能。帮助用户便捷地记录健康数据，并通过图表和统计分析了解变化趋势。

### 1.3 技术栈
- **前端**: React 18 + Material UI + Recharts
- **后端**: Go (Gin框架)
- **数据存储**: JSON文件存储（替代SQLite，因Windows环境编译限制）
- **认证**: JWT Token

## 2. 功能需求

### 2.1 用户管理
- 单用户模式，需要登录验证
- 登录功能（用户名/密码）
- 登出功能
- 支持修改密码
- 个人资料管理（姓名、年龄、身高、体重、性别）

### 2.2 血压记录管理

#### 2.2.1 记录字段
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 收缩压(高压) | 整数 | 是 | mmHg |
| 舒张压(低压) | 整数 | 是 | mmHg |
| 脉搏 | 整数 | 是 | 次/分钟 |
| 测量时间 | 日期时间 | 是 | 默认当前时间 |
| 用药 | 文本 | 否 | 记录是否服药及药物名称 |
| 备注 | 文本 | 否 | 其他说明 |

#### 2.2.2 操作功能
- 新增血压记录
- 查看记录列表（分页、排序）
- 编辑已有记录
- 删除记录（确认删除）
- 查看记录详情

### 2.3 血糖记录管理

#### 2.3.1 记录字段
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 血糖值 | 浮点数 | 是 | mmol/L |
| 测量类型 | 枚举 | 是 | 空腹/餐后2小时/随机/睡前 |
| 测量时间 | 日期时间 | 是 | 默认当前时间 |
| 餐食备注 | 文本 | 否 | 餐前/餐后相关备注 |
| 用药 | 文本 | 否 | 记录是否服药 |
| 备注 | 文本 | 否 | 其他说明 |

#### 2.3.2 操作功能
- 新增血糖记录
- 查看记录列表（分页、排序）
- 编辑已有记录
- 删除记录（确认删除）

### 2.4 用药管理

#### 2.4.1 用药计划
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 药品名称 | 文本 | 是 | 如：阿司匹林 |
| 剂量 | 文本 | 是 | 如：10mg |
| 单位 | 文本 | 否 | 片、粒、ml等 |
| 频率 | 枚举 | 是 | 每天/每天两次/每天三次/每周/自定义 |
| 服用时间 | 数组 | 否 | 如：["08:00", "20:00"] |
| 开始日期 | 日期 | 是 | 开始用药日期 |
| 用药天数 | 整数 | 否 | 0表示长期用药 |
| 备注 | 文本 | 否 | 其他说明 |

#### 2.4.2 服药打卡
- 今日用药进度展示
- 服药打卡记录
- 用药依从性统计

### 2.5 数据展示

#### 2.5.1 图表趋势
- 血压趋势折线图（显示收缩压、舒张压、脉搏）
- 血糖趋势折线图（显示血糖值变化）
- 支持选择时间范围（近7天、近30天、近90天、近1年）
- 鼠标悬停显示具体数值

#### 2.5.2 统计汇总
- 血压统计：平均值、最高值、最低值、健康等级
- 血糖统计：平均值、最高值、最低值、健康等级
- 用药统计：服药依从性

#### 2.5.3 健康提醒
- 根据血压值给出健康建议（采用中国标准）
- 根据血糖值给出健康建议（采用中国标准）
- 异常值醒目标识

### 2.6 界面要求
- 响应式设计，支持PC和移动端
- 简洁美观的用户界面
- 良好的用户体验

## 3. 非功能需求

### 3.1 性能要求
- 页面加载时间 < 2秒
- API响应时间 < 500ms

### 3.2 安全要求
- 密码加密存储（bcrypt）
- JWT Token认证
- API请求保护

### 3.3 兼容性
- 支持主流浏览器（Chrome、Firefox、Safari、Edge）
- 响应式布局支持移动端

## 5. API接口设计

### 5.1 认证接口
- `POST /api/auth/login` - 用户登录
- `PUT /api/auth/password` - 修改密码

### 5.2 个人资料接口
- `GET /api/profile` - 获取个人资料
- `PUT /api/profile` - 更新个人资料

### 5.3 血压记录接口
- `GET /api/records` - 获取记录列表
- `GET /api/records/:id` - 获取单条记录
- `POST /api/records` - 创建记录
- `PUT /api/records/:id` - 更新记录
- `DELETE /api/records/:id` - 删除记录

### 5.4 血糖记录接口
- `GET /api/sugar` - 获取记录列表
- `GET /api/sugar/:id` - 获取单条记录
- `POST /api/sugar` - 创建记录
- `PUT /api/sugar/:id` - 更新记录
- `DELETE /api/sugar/:id` - 删除记录

### 5.5 用药管理接口
- `GET /api/medication` - 获取用药计划
- `GET /api/medication/:id` - 获取单个用药计划
- `POST /api/medication` - 创建用药计划
- `PUT /api/medication/:id` - 更新用药计划
- `DELETE /api/medication/:id` - 删除用药计划
- `GET /api/medication/logs` - 获取服药记录
- `GET /api/medication/logs/today` - 今日用药进度
- `POST /api/medication/logs` - 服药打卡
- `GET /api/medication/stats` - 用药统计

### 5.6 统计接口
- `GET /api/stats/summary` - 血压统计汇总
- `GET /api/stats/trend` - 血压趋势数据
- `GET /api/sugar/stats/summary` - 血糖统计汇总
- `GET /api/sugar/stats/trend` - 血糖趋势数据

## 7. 健康标准参考

### 7.1 血压标准

| 分类 | 高压(收缩压) mmHg | 低压(舒张压) mmHg |
|------|-------------|-------------|
| 正常血压 | <120 | <80 |
| 正常高值 | 120-139 | 80-89 |
| 高血压1级(轻度) | 140-159 | 90-99 |
| 高血压2级(中度) | 160-179 | 100-109 |
| 高血压3级(重度) | ≥180 | ≥110 |

### 7.2 血糖标准（中国标准）

| 测量类型 | 正常范围 | 糖尿病前期 | 糖尿病 |
|---------|---------|-----------|--------|
| 空腹血糖 | 3.9-6.1 | 6.1-7.0 | ≥7.0 |
| 餐后2小时 | <7.8 | 7.8-11.1 | ≥11.1 |
